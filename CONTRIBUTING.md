# Contributing

This repository is a starter kit for future projects.

## Rules

- Use `main` as the default branch.
- Work on short-lived branches like `feature/*`, `fix/*`, and `chore/*`.
- Open a pull request for every meaningful change.
- Keep commits small and descriptive.
- Link issues to PRs whenever possible.

## Suggested workflow

1. Create or pick an issue.
2. Create a branch.
3. Implement the smallest useful change.
4. Open a PR using the template.
5. Verify before merging.
