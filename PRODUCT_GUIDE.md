# Guía para producto

Esta guía es para quien define valor, alcance y criterio de aceptación.

## Tu objetivo

- Convertir necesidades en trabajo bien definido.
- Mantener claros el problema, el valor y el alcance.
- Evitar ambigüedad para dev y QA.

## Flujo de trabajo

1. Abre un `epic` cuando el trabajo sea grande.
2. Divide el epic en `story` para cada parte de valor.
3. Usa `task` para trabajo interno cuando haga falta.
4. Define criterios de aceptación claros.
5. Revisa que cada item tenga prioridad y contexto suficientes.

## Qué debes cuidar

- La story debe explicar qué recibe el usuario.
- El epic debe explicar por qué existe el trabajo.
- El criterio de aceptación debe ser verificable.
- Si el alcance cambia, actualiza el issue antes de seguir.

## Qué no debes hacer

- No mezcles solución técnica con definición de valor si no es necesario.
- No dejes stories sin criterio de aceptación.
- No cambies alcance informalmente por chat sin actualizar el issue.
